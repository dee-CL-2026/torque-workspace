[2026-02-05 14:41:07] Claw Torque: [MEDIA: MessageMediaWebPage]
**Heartbeat 101:**

**What it is:**
A periodic "pulse" where OpenClaw wakes me up to check if anything needs attention. Like a cron job, but conversational.

**How it works:**

1. OpenClaw pings me every X minutes (configurable, e.g., every 30 min)
2. I receive a message: "Read [HEARTBEAT.md](http://heartbeat.md/)... do what it says"
3. I check `HEARTBEAT.md` for tasks
4. If nothing to do ‚Üí I reply `HEARTBEAT_OK` (silent, no notification to you)
5. If something needs attention ‚Üí I message you
[**HEARTBEAT.md**](http://heartbeat.md/)** is your control file:**

```# HEARTBEAT.md

- [ ] Check YouTube playlist for new videos
- [ ] Check email for urgent messages
- [ ] Any reminders due in next 2 hours?
```
Empty file = I do nothing, stay quiet.

**Why it's useful:**

‚Ä¢ I can be proactive without you asking
‚Ä¢ Background monitoring (inbox, calendars, playlists)
‚Ä¢ Batches multiple checks into one wake-up (cheaper than separate crons)
‚Ä¢ You control what I check by editing the file
**Heartbeat vs Cron:**

```| Heartbeat                    | Cron                           |
| ---------------------------- | ------------------------------ |
| One wake-up, multiple checks | One task per job               |
| Uses main session context    | Isolated sessions              |
| Flexible timing (~30 min)    | Exact timing (9:00 AM sharp)   |
| Cheaper for batched checks   | Better for scheduled reminders |
```
**To enable:**
We add tasks to `HEARTBEAT.md`, and the gateway handles the rest.

Want me to set it up with a starter checklist?
